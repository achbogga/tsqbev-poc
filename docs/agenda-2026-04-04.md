# Agenda: 2026-04-04 Foundation-Teacher Reset

This agenda replaces the old "teacher-region plus augmentation" agenda as the active forward plan.

## Why This Agenda Exists

Recent evidence says the local sparse student is no longer failing for one simple reason. The repo
already recovered a stronger mini frontier on the current winner line, but the last steps have been
incremental rather than decisive. At the same time, public evidence says:

- `Sparse4Dv3` with strong image priors is a stronger camera-only temporal frontier than treating
  BEVFusion as the final student direction
- `BEVFormer v2` shows perspective supervision is a direct fix for adapting modern image backbones
  to 3D BEV reasoning
- `DINOv2` and `DINOv3` give us foundation features and public weights now
- the unrestricted teacher can remain much larger than the student

That combination is the next high-ROI path to a materially better student.

## Primary Direction

The active primary direction is now:

1. `Sparse4D`-style sparse temporal student
2. `BEVFormer v2`-style perspective supervision
3. `DINOv2` feature projection first, `DINOv3` second
4. `OpenPCDet` / `BEVFusion` teacher supervision in BEV, box, and quality space
5. `MapTRv2` lane/vector-map head only after the isolated lane baseline is real

## Immediate Sequence

### Branch D1: Finish the current winner-line continuation

- let the current `v29` continuation finish
- keep it only if it materially beats `v28`
- if it stalls or regresses, stop all further weight-only nudges on this family

### Branch D2: Foundation Camera Projector

- add a frozen `DINOv2` camera feature path projected into the student camera branch
- keep the runtime student lightweight; the projector must be a small linear or `1x1` adapter
- use activation checkpointing and GPU auto-fit if needed

### Branch D3: Perspective Supervision

- add a `BEVFormer v2`-style auxiliary perspective head
- use it to train the camera branch to encode 3D cues before BEV / sparse aggregation
- the perspective branch must be removable at export if it is training-only

### Branch D4: Teacher Upgrade

- keep `OpenPCDet` / `BEVFusion` as dense geometry teachers
- add stronger teacher-output supervision for BEV or proposal quality maps
- keep `Alpamayo` scoped to hard-case mining, evaluation, or scenario critique, not the runtime

## Lane Branch

Lane is now allowed to move in parallel, but only under non-regression discipline.

### Branch L1: OpenLane isolated warm-start

- run the queued OpenLane V1 warm-start
- verify real losses, export, and evaluator wiring

### Branch L2: Shared-latent lane integration

- only after L1 is real
- add a `MapTRv2`-style vector head to the same latent used by detection
- enforce detection non-regression on every joint run

## Stop Conditions

- if `v29` and the next continuation both fail to beat `v28`, stop winner-line weight nudges
- if the DINO projector path does not produce cleaner validation or better NDS than the current
  line within two bounded runs, reassess the camera foundation choice
- if lane-only OpenLane is not healthy, do not open joint detection+lane
