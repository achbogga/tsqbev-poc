## Active Detection Branch

This remains the main research loop until the current sparse student clears the mini scale gate or
the dense-BEV reset displaces it.

### Branch D1: Teacher-Region Ranking

- Objective:
  - preserve the `v20` reduction in boxes/sample
  - improve `NDS/mAP` by supervising query objectness with a soft teacher-region target instead of
    relying only on matched-query losses and anchor priors
- Why:
  - DistillBEV shows that region-aware distillation is high ROI when teacher geometry is strong but
    student focus is diffuse
  - our current teacher cache already has boxes and scores, so region supervision is available now
    without waiting for dense teacher maps
- Current implementation:
  - low-weight teacher-region objectness around cached teacher boxes
  - radius-weighted soft targets from teacher score and BEV distance

### Branch D2: Label-Safe Robustness Augmentation

- Objective:
  - improve mini-val robustness without introducing new label-transform bugs
- Why:
  - BEVFusion and related stacks rely on image + 3D augmentation, but tonight's highest-ROI safe
    slice is still the label-preserving part
- Current implementation:
  - multiview photometric distortion
  - blur / noise / random erasing
  - LiDAR dropout / jitter
- Not in scope tonight:
  - camera/LiDAR synchronized geometric flips or rotations
  - any transform that requires box, lane, calibration, or proposal rewrites

### Detection Stop Conditions

- Stop this sparse-student branch if either condition is reached:
  - two bounded mini runs fail to beat `v16` while already carrying the `v20` box-count reduction
  - the dense-BEV reset control arm produces a compact student path that is clearly better grounded
    and easier to scale

## Lane Branch

Lane work is now allowed to move forward, but it remains a separate bounded track until the
detection frontier is healthier.

### Branch L1: OpenLane V1 Sanity Baseline

- Objective:
  - verify that the existing lane head trains on real OpenLane V1 data, logs sane losses, and can
    export predictions cleanly
- Why:
  - the dataset path is now ready
  - we need a real lane-side control before attempting any detection+lane mixture
- Scope:
  - bounded `lane3d_300` training subset
  - single-view pretrained image backbone
  - moderate image augmentation only

### Branch L2: Lane Evaluation Wiring

- Objective:
  - turn the lane baseline from a training-only smoke into a measured artifact
- Why:
  - lane must become evidence-backed before it enters the main agenda
- Scope:
  - export predictions
  - run the official OpenLane evaluator
  - publish a lane-side summary artifact and docs update

### Lane Rules

- Do not merge lane into the main `nuScenes mini` detection loop yet.
- Do not claim multitask lift until both of these are true:
  - the lane-only baseline works on real OpenLane data
  - the detection incumbent is no longer bottlenecked by ranking/source-mix collapse
